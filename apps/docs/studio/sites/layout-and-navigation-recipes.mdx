## Sidebar + Dropdown (default)

- Best for deep hierarchies and multiple spaces
- Per‑space dropdowns keep the sidebar compact

## Tabs layout

- Best for 2–4 spaces of similar importance
- Clear, top‑level switching between spaces

## Space order

- Put your primary user guide or “Getting started” first
- Follow with API, integrations, admin guides, etc.

## Buttons

- Add “Changelog”, “Status”, or “Contact” buttons where users expect them
- Use Topbar (Right) for external links; Sidebar (Bottom) for utility links
