## Before you begin

- Account created and signed in
- You belong to (or created) an organization
- Role permissions: Member can write; Admin/Owner can create sites, publish, and manage domains

## Create a space

1. Open the sidebar → Spaces → Create space
2. Name it (for example, “User Guide”), optionally pick an icon and description

## Add your first page

1. Inside the space, click New page
2. Type a title, then start writing in the editor
3. Type `/` to open the slash menu for headings, lists, code blocks, tables, tabs, etc.

## (Optional) Import API reference

1. Add a document of type “API Spec” (from the space tree actions)
2. Paste your OpenAPI 3.0/3.1 JSON or YAML and click Upload, then Process
3. Docufy generates endpoint pages and tag groups under the spec

<img src="https://izfvzcggescmuprs.public.blob.vercel-storage.com/assets/docufy/963312bb-c99a-4c53-ab45-c6fc1bc1f1b7/136af403-e540-4c5d-bd63-3edfa63453df-Untitled-6.png" />

## Create a site and select spaces

1. Sidebar → Sites → Create site
2. Name and save; open the site page
3. In Content, add one or more spaces and order them

## Brand and theme

1. In Appearance, upload light/dark logos and a favicon
2. Open Theme Studio to adjust brand colors, surfaces, and layout tokens

## Publish

1. Go to Deploys → Publish
2. Watch the build progress and then click the primary host link to view your docs
3. Blazing fast customer facing documentation is ready!

<img src="https://izfvzcggescmuprs.public.blob.vercel-storage.com/assets/docufy/963312bb-c99a-4c53-ab45-c6fc1bc1f1b7/46fe02e4-2256-4bbf-a8d6-fea24b208b86-Untitled-7.png" />

## Add a custom domain

1. Open Domains → Add domain (e.g., [docs.example.com](http://docs.example.com))
2. Follow the DNS instructions shown for your domain type (CNAME for subdomains; A/AAAA for apex)
3. Click Check now to verify; SSL is issued automatically after DNS propagates
4. Republish the site after verification is completed
